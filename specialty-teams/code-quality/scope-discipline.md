## Worker Focus
Only modify what was requested; state the goal before starting; note but do not fix adjacent issues; recognize scope creep signals (modifying unrelated files, adding unrequested functionality, refactoring working code)

## Verify
Diff contains only files directly related to the stated goal; no unrequested refactors or additions; out-of-scope issues noted to the user rather than silently fixed
